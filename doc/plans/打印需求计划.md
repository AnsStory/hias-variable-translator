# 打印需求计划

## 功能概述

### 功能1：插入 console.log

选中文件中的部分内容（支持多选），按下快捷键 `Ctrl+Alt+L` 后，在下一行添加 `console.log` 语句。

支持 snippet 语法（光标占位符），可自定义光标跳转位置。

### 功能2：批量管理 console.log

- `Ctrl+Alt+D` - 删除文件中与配置格式一致的 console.log
- `Ctrl+Alt+C` - 注释文件中与配置格式一致的 console.log
- `Ctrl+Alt+U` - 取消注释文件中与配置格式一致的 console.log

**重要**：仅处理与配置的 `consoleLogTemplate` 格式一致的 console.log，不一致的不进行处理。

## 配置项

### 配置项说明

| 配置键 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `enableConsoleLog` | boolean | `false` | 控制是否启用 console.log 快速插入功能 |
| `consoleLogTemplate` | string | `` `${value}, \`${name}\`` `` | console.log 参数模板 |

### 模板变量语法

#### 基础变量

| 语法 | 说明 | 示例 | 结果 |
|------|------|------|------|
| `${value}` | 原始文本 | `obj` | `obj` |
| `${name}` | 原始文本（不添加引号） | `obj` | `obj` |
| `${name:prefix,suffix}` | 前缀+原始文本+后缀 | `obj` | `prefixobjsuffix` |

#### Snippet 语法（光标占位符）

| 语法 | 说明 |
|------|------|
| `$1`, `$2`, ... `$9` | 光标位置，按 Tab 依次跳转 |
| `$0` | 最终光标位置 |
| `${1:placeholder}` | 带默认值的占位符，选中后可直接编辑 |

### 模板示例

#### 普通模式（不带 snippet）

| 模板 | 选中 `obj` 时输出 |
|------|-------------------|
| `` `${value}, \`${name}\` `` | `console.log(obj, \`obj\`)` (反引号) |
| `${value}, '${name}'` | `console.log(obj, 'obj')` (单引号) |
| `${value}, "${name}"` | `console.log(obj, "obj")` (双引号) |
| `${value}` | `console.log(obj)` |

#### Snippet 模式（带光标占位符）

| 模板 | 说明 |
|------|------|
| `` `${value}, \`${1:name}\`$0 `` | 插入后光标停在 `name` 位置，按 Tab 跳到最后 |
| `` `$1${value}$2, \`$1${name}\`$0 `` | 多光标跳转 |
| `` console.log($1${value}, `$2${name}`$0) `` | 完整 snippet |

### 配置示例

```json
// 普通模式
"variableTranslator.consoleLogTemplate": "${value}, `${name}`"

// Snippet 模式：插入后光标停在 name 位置
"variableTranslator.consoleLogTemplate": "${value}, `${1:name}`$0"

// Snippet 模式：插入后光标停在 value 位置
"variableTranslator.consoleLogTemplate": "${1:value}, `${name}`$0"
```

### Snippet 模式说明

- 如果模板中包含 `$1`、`${1:...}` 等语法，则启用 snippet 模式
- **单选和多选均支持 snippet 模式**
- 使用 `WorkspaceEdit` + `SnippetTextEdit` API 实现多位置 snippet 插入
- 插入后按 Tab 键可在各光标位置间跳转

### 功能2：批量管理 console.log

#### 删除 console.log（Ctrl+Alt+D）

```javascript

console.log(obj, `obj`)  // 匹配模板，删除

// 执行删除后

```

#### 注释 console.log（Ctrl+Alt+C）

```javascript
// 原始代码
const obj = { name: '张三' }
console.log(obj, `obj`)  // 匹配模板，注释

// 执行注释后
const obj = { name: '张三' }
// console.log(obj, `obj`)  // 匹配模板，已注释
console.log('其他', x)       // 不匹配，保留
// console.log(obj, `obj`)  // 匹配模板，已注释
```

#### 取消注释 console.log（Ctrl+Alt+U）

```javascript

console.log('其他', x)       // 不匹配，保留

// 执行取消注释后
const obj = { name: '张三' }
console.log(obj, `obj`)  // 已取消注释
console.log('其他', x)   // 不匹配，保留
console.log(obj, `obj`)  // 已取消注释
```

## 技术实现

### 1. 插入 console.log

#### 核心逻辑

1. 获取选中的文本内容（支持多选）
2. 读取模板配置
3. 判断是否启用 snippet 模式
4. 解析模板变量（`${value}`、`${name}`、`${name:prefix,suffix}`）
5. 使用 AST 解析确定正确的插入位置
6. 根据模式插入：
   - **Snippet 模式**：使用 `WorkspaceEdit` + `SnippetTextEdit` 支持多位置 snippet
   - **普通模式**：使用 `editor.edit()` 直接插入文本
7. 处理缩进（保持与当前行相同的缩进）

#### 插入位置计算（参考 turbo-console-log）

使用 `acorn` + `@sveltejs/acorn-typescript` 解析代码生成 AST，根据选中变量的上下文确定插入位置：

| 代码类型 | 插入位置 |
|----------|----------|
| 普通赋值 `let x = 1` | 下一行 |
| 对象字面量 `let x = {...}` | 对象结束后下一行 |
| 数组赋值 `let x = [...]` | 数组结束后下一行 |
| 函数赋值 `let x = function() {...}` | 函数结束后下一行 |
| 函数调用 `let x = foo()` | 函数调用结束后下一行 |
| 函数调用赋值 `let x = ref({...})` | 函数调用结束后下一行 |
| 三元表达式 `let x = a ? b : c` | 下一行 |
| 属性访问 `let x = obj.prop` | 下一行 |

#### AST 解析流程

```
1. 获取文件扩展名
2. 判断是否需要提取 script（Vue/HTML）
3. 使用 acorn + @sveltejs/acorn-typescript 解析代码
4. 调整 AST 位置（如果有 script 偏移）
5. 遍历 AST 找到变量声明
6. 判断初始化表达式类型：
   - 包含函数调用 → 使用字节偏移计算结束位置
   - 对象字面量 → 使用对象结束行
   - 数组 → 使用数组结束行
   - 其他 → 使用声明结束行
7. 返回插入行号
```

#### Snippet 插入实现

```typescript
// Snippet 模式：使用 WorkspaceEdit + SnippetTextEdit
const workspaceEdit = new vscode.WorkspaceEdit()
const snippetEdits: vscode.SnippetTextEdit[] = []

for (const insertion of insertions) {
  const snippetString = new vscode.SnippetString(snippetText)
  snippetEdits.push(vscode.SnippetTextEdit.insert(range.start, snippetString))
}

workspaceEdit.set(editor.document.uri, snippetEdits)
await vscode.workspace.applyEdit(workspaceEdit)
```

### 2. 批量管理 console.log

#### 核心逻辑

1. 读取整个文件内容
2. 根据配置的模板格式，构建匹配正则表达式
3. 遍历每一行，检查是否匹配模板格式
4. 根据命令执行相应操作：
   - **删除**：移除匹配的行
   - **注释**：在行首添加 `//`
   - **取消注释**：移除行首的注释符号
5. 应用修改到文件

#### 匹配规则

根据 `consoleLogTemplate` 配置构建正则表达式：

| 模板 | 匹配正则 |
|------|----------|
| `` `${value}, \`${name}\`` `` | `/console\.log\(.+,\s*`.+`\)$/` |
| `'前置', ${value}, '后置'` | `/console\.log\('前置',\s*.+,\s*'后置'\)$/` |
| `${value}` | `/console\.log\(.+\)$/` |

## 代码结构

### 文件结构

```
src/
├── print/
│   ├── handler.ts    # console.log 插入、删除、注释、取消注释处理函数
│   ├── config.ts     # 配置管理、模板解析、正则构建
│   ├── ast.ts        # AST 解析，确定插入位置（支持 Vue/HTML/JSX/TSX）
│   └── index.ts      # 模块导出
├── extension.ts      # 命令注册
└── translate/        # 翻译模块（原有功能）
```

### 依赖项

| 依赖 | 用途 |
|------|------|
| `acorn` | JavaScript/TypeScript 代码解析 |
| `@sveltejs/acorn-typescript` | TypeScript + JSX/TSX 支持 |
| `@types/acorn` | acorn 类型定义 |

### 核心函数

**handler.ts**
- `handleInsertConsoleLog()` - 插入 console.log（支持多选和 snippet）
- `handleDeleteConsoleLog()` - 批量删除 console.log
- `handleCommentConsoleLog()` - 批量注释 console.log
- `handleUncommentConsoleLog()` - 批量取消注释 console.log

**config.ts**
- `parseConsoleLogTemplate(value, template)` - 解析模板变量
- `hasSnippetSyntax(template)` - 检查模板是否包含 snippet 语法
- `cleanSnippetSyntax(text)` - 清理 snippet 语法
- `buildConsoleLogRegex(template)` - 构建匹配正则表达式

**ast.ts**
- `parseCode(code, selectionLine, fileExtension)` - 解析代码生成 AST
- `findInsertionLine(code, variableName, selectionLine, fileExtension)` - 使用 AST 确定插入位置
- `extractVueScript(sourceCode, selectionLine)` - 提取 Vue 文件中的 script 内容
- `adjustASTLocations(node, lineOffset, byteOffset)` - 调整 AST 位置偏移

## 注意事项

- **功能开关**：默认禁用，需在设置中开启 `enableConsoleLog`
- **缩进处理**：自动读取当前行的缩进，并应用到新插入的行
- **多选支持**：支持同时选中多个变量，为每个变量插入对应的 console.log
- **Snippet 支持**：单选和多选均支持 snippet 语法，可自定义光标跳转位置
- **模板灵活性**：用户可自定义输出格式，支持反引号、单引号、双引号包裹
- **匹配准确性**：仅处理与配置模板格式一致的 console.log，不匹配的保留不变
- **边界情况**：支持文件末尾插入、对象字面量结束后插入、函数调用结束后插入等复杂场景
- **文件类型支持**：支持 JS/TS/Vue/HTML/JSX/TSX 文件
