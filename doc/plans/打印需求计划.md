# 打印需求计划

> **当前版本**：v0.1.14 | **状态**：已完成 ✅

## 功能概述

### 功能1：插入 console.log

选中文件中的部分内容（支持多选），按下快捷键 `Ctrl+Alt+L` 后，在下一行添加 `console.log` 语句。

支持 snippet 语法（光标占位符），可自定义光标跳转位置。

### 功能2：批量管理 console.log

- `Ctrl+Alt+D` - 删除文件中与配置格式一致的 console.log
- `Ctrl+Alt+C` - 注释文件中与配置格式一致的 console.log
- `Ctrl+Alt+U` - 取消注释文件中与配置格式一致的 console.log

**重要**：仅处理与配置的 `consoleLogTemplate` 格式一致的 console.log，不一致的不进行处理。

### 功能3：插入后自动复制到剪贴板

插入 console.log 后，自动将模板中指定的部分复制到剪贴板。

使用与 `consoleLogTemplate` 相同的变量语法配置要复制的内容。

## 配置项

### 配置项说明

| 配置键 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `enableConsoleLog` | boolean | `false` | 控制是否启用 console.log 快速插入功能 |
| `consoleLogTemplate` | string | `` `${value}, \`${name}\`` `` | console.log 参数模板 |
| `consoleLogCopyToClipboard` | boolean | `false` | 插入 console.log 后是否自动将匹配内容复制到剪贴板 |
| `consoleLogClipboardPattern` | string | `""` | 剪贴板提取模板，使用与 consoleLogTemplate 相同的语法 |

### 模板变量语法

#### 基础变量

| 语法 | 说明 | 示例 | 结果 |
|------|------|------|------|
| `${value}` | 原始文本 | `obj` | `obj` |
| `${name}` | 原始文本（不添加引号） | `obj` | `obj` |
| `${name:prefix,suffix}` | 前缀+原始文本+后缀 | `obj` | `prefixobjsuffix` |

#### Snippet 语法（光标占位符）

| 语法 | 说明 |
|------|------|
| `$1`, `$2`, ... `$9` | 光标位置，按 Tab 依次跳转 |
| `$0` | 最终光标位置 |
| `${1:placeholder}` | 带默认值的占位符，选中后可直接编辑 |

#### Snippet + 变量组合

| 语法 | 说明 | 示例 |
|------|------|------|
| `${1:value}` | snippet + 选中文本 | 插入后光标停在 value 位置 |
| `${1:name}` | snippet + 变量名 | 插入后光标停在 name 位置 |
| `${1:name:前,后}` | snippet + 变量名 + 前后缀 | 插入后光标停在 前obj后 位置 |

### 模板示例

#### 普通模式（不带 snippet）

| 模板 | 选中 `obj` 时输出 |
|------|-------------------|
| `` `${value}, \`${name}\` `` | `console.log(obj, \`obj\`)` (反引号) |
| `${value}, '${name}'` | `console.log(obj, 'obj')` (单引号) |
| `${value}, "${name}"` | `console.log(obj, "obj")` (双引号) |
| `${value}` | `console.log(obj)` |

#### Snippet 模式（带光标占位符）

| 模板 | 说明 |
|------|------|
| `` `${value}, \`${1:name}\`$0 `` | 插入后光标停在 `name` 位置，按 Tab 跳到最后 |
| `` `$1${value}$2, \`$1${name}\`$0 `` | 多光标跳转 |
| `` console.log($1${value}, `$2${name}`$0) `` | 完整 snippet |

### 配置示例

```json
// 普通模式
"variableTranslator.consoleLogTemplate": "${value}, `${name}`"

// Snippet 模式：插入后光标停在 name 位置
"variableTranslator.consoleLogTemplate": "${value}, `${1:name}`$0"

// Snippet 模式：插入后光标停在 value 位置
"variableTranslator.consoleLogTemplate": "${1:value}, `${name}`$0"


// 启用剪贴板复制：复制 log_ 前缀后的变量名
"variableTranslator.consoleLogTemplate": "${value}$1, `${name:log_,}`$0",
"variableTranslator.consoleLogCopyToClipboard": true
"variableTranslator.consoleLogClipboardPattern": "${name:log_,}"
```

### Snippet 模式说明

- 如果模板中包含 `$1`、`${1:...}` 等语法，则启用 snippet 模式
- **单选和多选均支持 snippet 模式**
- 使用 `WorkspaceEdit` + `SnippetTextEdit` API 实现多位置 snippet 插入
- 插入后按 Tab 键可在各光标位置间跳转

## 技术实现

### 1. 插入 console.log

#### 核心逻辑

1. 获取选中的文本内容（支持多选）
2. 读取模板配置
3. 判断是否启用 snippet 模式
4. 解析模板变量（`${value}`、`${name}`、`${name:prefix,suffix}`）
5. 使用 AST 解析确定正确的插入位置
6. 根据模式插入：
   - **Snippet 模式**：使用 `WorkspaceEdit` + `SnippetTextEdit` 支持多位置 snippet
   - **普通模式**：使用 `editor.edit()` 直接插入文本
7. 处理缩进（保持与当前行相同的缩进）

#### 插入位置计算（参考 turbo-console-log，支持 16 种场景）

使用 `acorn` + `@sveltejs/acorn-typescript` 解析代码生成 AST，根据选中变量的上下文确定插入位置：

| # | 场景 | 代码示例 | 插入位置 |
|---|------|----------|----------|
| 1 | 命名函数赋值 | `const myFn = function() {}` | 函数结束后下一行 |
| 2 | 函数参数 | `function greet(name) {}` | 函数体开始处 |
| 3 | 对象方法调用赋值 | `const result = obj.method()` | 函数调用结束后下一行 |
| 4 | 函数调用赋值 | `const result = doSomething()` | 函数调用结束后下一行 |
| 5 | 属性访问赋值 | `const value = obj.prop` | 声明结束后下一行 |
| 6 | 对象字面量 | `const config = { key: value }` | 对象结束后下一行 |
| 7 | 数组赋值 | `const items = [1, 2, 3]` | 数组结束后下一行 |
| 8 | 模板字符串 | `` const msg = `Hello ${name}` `` | 表达式结束后下一行 |
| 9 | 三元表达式 | `const x = condition ? a : b` | 表达式结束后下一行 |
| 10 | 二元/逻辑表达式 | `const sum = a + b` | 声明结束后下一行 |
| 11 | 属性方法调用 | `obj.method()` | 方法调用结束后下一行 |
| 12 | 原始属性访问 | `const obj = { name: 'value' }` | 声明结束后下一行 |
| 13 | 基本类型赋值 | `const x = 42` | 声明结束后下一行 |
| 14 | 条件块内 | `if (variable) {}` | 条件语句之前 |
| 15 | return 语句内 | `return variableName` | return 语句之前 |
| 16 | 游离表达式（兜底） | 任意表达式 | 表达式结束后下一行 |

#### TypeScript 特殊语法支持

| 特性 | 说明 |
|------|------|
| `as` 类型断言 | `const obj = fn() as Type` → 自动解包 |
| `<Type>` 类型断言 | `const obj = <Type>fn()` → 自动解包 |
| 括号表达式 | `const obj = (fn())` → 自动解包 |
| 非空断言 `!` | `const obj = fn()!` → 自动解包 |
| 可选链 `?.` | `const value = obj?.prop` → 支持 |
| ChainExpression | `const value = obj?.method()` → 自动解包 |
| 解构赋值 | `const { data } = fn()` → 支持 |
| 类属性定义 | `protected config = new Foo()` → 支持 |
| TS 参数属性 | `constructor(private name: string)` → 支持 |
| 私有字段 | `#private` → 支持 |
| 标签模板 | `` html`<div/>` `` → 支持 |
| 箭头函数表达式体 | `() => expr` → 返回转换提示 |

#### 场景检测优先级

按优先级从高到低依次匹配，确保更具体的模式优先：

| 优先级 | 场景 |
|--------|------|
| 1 | 命名函数赋值 |
| 2 | 函数参数 |
| 3 | 对象方法调用赋值 |
| 4 | 函数调用赋值 |
| 5 | 属性访问赋值 |
| 6 | 对象字面量 |
| 7 | 数组赋值 |
| 8 | 模板字符串 |
| 9 | 三元表达式 |
| 10 | 二元/逻辑表达式 |
| 11 | 属性方法调用 |
| 12 | 原始属性访问 |
| 13 | 基本类型赋值 |
| 14 | 条件块内 |
| 15 | return 语句内 |
| 16 | 游离表达式（兜底） |

#### AST 解析流程

```
1. 获取文件扩展名
2. 判断是否需要提取 script（Vue/HTML/Svelte/Astro）
3. 使用 acorn + @sveltejs/acorn-typescript 解析代码
4. 调整 AST 位置（如果有 script 偏移）
5. 按优先级遍历检查 16 种场景：
   - 命名函数赋值 → 函数结束后
   - 函数参数 → 函数体开始处
   - 函数调用赋值 → 字节偏移计算结束位置
   - 属性访问赋值 → 声明结束后
   - 对象字面量 → 对象结束后
   - 数组 → 数组结束后
   - 属性方法调用 → 调用结束后（排除 return 上下文）
   - 基本类型赋值 → 声明结束后（for 循环特殊处理）
   - 其他 → 声明结束后
6. 返回插入行号和转换信息
```

#### Snippet 插入实现

```typescript
// Snippet 模式：使用 WorkspaceEdit + SnippetTextEdit
const workspaceEdit = new vscode.WorkspaceEdit()
const snippetEdits: vscode.SnippetTextEdit[] = []

for (const insertion of insertions) {
  const snippetString = new vscode.SnippetString(snippetText)
  snippetEdits.push(vscode.SnippetTextEdit.insert(range.start, snippetString))
}

workspaceEdit.set(editor.document.uri, snippetEdits)
await vscode.workspace.applyEdit(workspaceEdit)
```

### 2. 批量管理 console.log

#### 核心逻辑

1. 读取整个文件内容
2. 根据配置的模板格式，构建匹配正则表达式
3. 遍历每一行，检查是否匹配模板格式
4. 根据命令执行相应操作：
   - **删除**：移除匹配的行
   - **注释**：在行首添加 `//`
   - **取消注释**：移除行首的注释符号
5. 应用修改到文件

#### 匹配规则

根据 `consoleLogTemplate` 配置构建正则表达式：

| 模板 | 匹配正则 |
|------|----------|
| `` `${value}, \`${name}\`` `` | `/console\.log\(.+,\s*`.+`\)$/` |
| `'前置', ${value}, '后置'` | `/console\.log\('前置',\s*.+,\s*'后置'\)$/` |
| `${value}` | `/console\.log\(.+\)$/` |

## 注意事项

- **功能开关**：默认禁用，需在设置中开启 `enableConsoleLog`
- **缩进处理**：自动读取当前行的缩进，并应用到新插入的行
- **多选支持**：支持同时选中多个变量，为每个变量插入对应的 console.log
- **Snippet 支持**：单选和多选均支持 snippet 语法，可自定义光标跳转位置
- **模板灵活性**：用户可自定义输出格式，支持反引号、单引号、双引号包裹
- **匹配准确性**：仅处理与配置模板格式一致的 console.log，不匹配的保留不变
- **场景覆盖**：支持 16 种代码场景，与 turbo-console-log 功能完全对齐
- **TypeScript 支持**：支持类型断言、非空断言、可选链、参数属性等特殊语法
- **文件类型支持**：支持 JS/TS/Vue/HTML/JSX/TSX/Svelte/Astro 文件
- **箭头函数转换**：箭头函数表达式体场景会返回转换提示，调用方需转为 block body

### 用户配置示例（settings.json）

```json
{
  // 启用 console.log 功能
  "variableTranslator.enableConsoleLog": true,

  // ========== 普通模式配置 ==========

  // 默认配置：反引号包裹
  "variableTranslator.consoleLogTemplate": "${value}, `${name}`"

  // 单引号包裹
  "variableTranslator.consoleLogTemplate": "${value}, '${name}'"

  // 双引号包裹
  "variableTranslator.consoleLogTemplate": "${value}, \"${name}\""

  // 带前后缀
  "variableTranslator.consoleLogTemplate": "${value}, ${name:前缀,后缀}"

  // ========== Snippet 模式配置 ==========

  // 插入后光标停在 name 位置
  "variableTranslator.consoleLogTemplate": "${value}, `${1:name}`$0"

  // 插入后光标停在 value 位置
  "variableTranslator.consoleLogTemplate": "${1:value}, `${name}`$0"

  // 多光标跳转
  "variableTranslator.consoleLogTemplate": "'❁', ${value}$1, `${name:前,后}`$0"

  // ========== 剪贴板复制配置 ==========

  // 启用剪贴板复制功能
  "variableTranslator.consoleLogCopyToClipboard": true,

  // 复制 log_ 前缀后的变量名（如选中 params，复制 params）
  "variableTranslator.consoleLogClipboardPattern": "${name:log_,}",

  // 复制整个带反引号的 name（如选中 params，复制 `params`）
  "variableTranslator.consoleLogClipboardPattern": "${name:`",

  // 复制选中的原始文本
  "variableTranslator.consoleLogClipboardPattern": "${value}"
}
```

### 配置说明

| 配置键 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `enableConsoleLog` | boolean | `false` | 启用/禁用 console.log 插入功能 |
| `consoleLogTemplate` | string | `${value}, \`${name}\`` | console.log 参数模板 |
| `consoleLogCopyToClipboard` | boolean | `false` | 插入后是否自动复制到剪贴板 |
| `consoleLogClipboardPattern` | string | `""` | 剪贴板提取模板，使用与 consoleLogTemplate 相同的语法 |

### 模板语法速查

| 语法 | 说明 | 示例 |
|------|------|------|
| `${value}` | 选中的原始文本 | `obj` |
| `${name}` | 原始文本（不添加引号） | `obj` |
| `${name:前,后}` | 前缀+原始文本+后缀 | `前obj后` |
| `$1`, `$2`, ... | snippet 光标位置 | 按 Tab 跳转 |
| `$0` | 最终光标位置 | 最后停留 |
| `${1:placeholder}` | 带默认值的占位符 | 选中可编辑 |

> **注意**：`consoleLogClipboardPattern` 使用与 `consoleLogTemplate` 相同的变量语法，但不支持 snippet 语法（`$1`、`$0` 等）
