# 变量翻译助手 - VSCode 插件

> **当前版本**：v0.1.10 | **状态**：已完成 ✅

## 功能概述

一款 VSCode 插件，自动检测并翻译任意非英文字符为英文（中文、日文、韩文、俄文等），支持文件路径翻译和选中文本翻译。

---

## 功能1：新建/重命名文件时翻译路径

### 使用场景
用户在 VSCode 资源管理器中右键新建文件或文件夹，或重命名现有文件/文件夹，输入包含任意非英文字符（中日韩俄等）的路径，插件自动检测语言并翻译为英文。

### 操作流程
```
新建：右键新建文件/文件夹 → 输入路径 → 回车 → 选择翻译格式 → 回车 → 自动翻译创建
重命名：右键重命名文件/文件夹 → 输入新名称 → 回车 → 选择翻译格式 → 回车 → 自动翻译
```

### 示例
```
输入：用户名称/测试文件/测试.test.js

格式选择：
- camelCase:      userName/testFile/test.test.js
- PascalCase:     UserName/TestFile/Test.test.js
- snake_case:     user_name/test_file/test.test.js
- CONSTANT_CASE:  USER_NAME/TEST_FILE/TEST.test.js
- param-case:     user-name/test-file/test.test.js
- Header-Case:    User-Name/Test-File/Test.test.js
- noCase:         user name /test file /test.test.js
- noCaseCamel:    userName/testFile/test.test.js
```

### 取消操作行为
```
新建文件：按 ESC 取消 → 删除新建的文件/文件夹
重命名文件：按 ESC 取消 → 保留原文件/文件夹（不删除）
```

### 撤回功能
```
按 Alt+Shift+Z → 删除翻译后的文件/目录 → 关闭该文件的编辑器窗口
```

**撤回规则**：
- 撤回操作会直接删除翻译后的文件，而不是恢复到原始非英文路径
- 只关闭被删除文件的编辑器窗口，不会关闭其他已打开的文件
- **1分钟内可撤回**，1分钟后自动清空撤回缓存

### 文件名冲突处理
```
场景：目标文件已存在（如 test.test.js 已存在）
处理：自动添加后缀（如 test_1.test.js）并提示用户
```

### 翻译失败降级
```
场景：翻译API调用失败（网络错误、配额用完等）
处理：自动降级为拼音翻译
```

### 配置项
此功能可在设置中开启/关闭：
```json
{
  "variableTranslator.enableFileTranslation": true
}
```

## 功能2：编辑器中选中文本翻译

### 使用场景
用户在任意文件中编写非英文字符（中日韩俄等），选中后通过快捷键自动检测语言并翻译替换为英文。

### 操作流程
```
选中文本 → 按下 Alt+Shift+T → 选择翻译格式 → 回车 → 替换为英文
```

### 示例
```
选中：用户名称
按下：Alt+Shift+T

格式选择：
- camelCase:      userName
- PascalCase:     UserName
- snake_case:     user_name
- CONSTANT_CASE:  USER_NAME
- param-case:     user-name
- Header-Case:    User-Name
- Capital Case:   User Name
- no case:        user name
```

### 撤回功能
使用 VSCode 自带的撤销功能：`Ctrl+Z`

## 命名格式说明

### 文件翻译格式

| 格式 | 示例 | 说明 |
|------|------|------|
| camelCase | userName | 小驼峰，首字母小写 |
| PascalCase | UserName | 大驼峰，首字母大写 |
| snake_case | user_name | 下划线分隔，全小写 |
| CONSTANT_CASE | USER_NAME | 下划线分隔，全大写 |
| param-case | user-name | 连字符分隔，全小写 |
| Header-Case | User-Name | 连字符分隔，首字母大写 |

### 选中文本翻译额外格式

| 格式 | 示例 | 说明 |
|------|------|------|
| Capital Case | User Name | 首字母大写，空格分隔 |
| no case | user name | 全小写，空格分隔 |

| 快捷键 | 功能 | 说明 |
|--------|------|------|
| `Alt+Shift+T` | 翻译选中文本 | 编辑器中选中非英文文本后翻译 |

## 翻译服务支持

### 服务列表
| 服务 | 认证方式 | 费用 | 说明 |
|------|----------|------|------|
| Pinyin | 零配置 | 免费 | 降级方案，无需配置 |
| ChatGPT / OpenAI | API Key | 按量付费 | 需要 OpenAI API Key |
| 谷歌翻译 | 免费（有限制） | 免费 | 无需配置，但有调用限制 |
| Bing / Azure Translator | API Key | 按量付费 | 需要 Azure 账号 |
| DeepLX | 本地部署 | 免费 | 需要本地部署服务 |
| 百度翻译 | APP_ID + Key | 按量付费 | 需要百度翻译开放平台账号 |
| 腾讯翻译君 | SecretId + SecretKey | 按量付费 | 需要腾讯云账号 |

### API 获取方式
| 服务 | 获取地址 | 说明 |
|------|----------|------|
| OpenAI | [platform.openai.com/api-keys](https://platform.openai.com/api-keys) | 创建 API Key |
| 百度翻译 | [fanyi-api.baidu.com](https://fanyi-api.baidu.com/) | 开通通用翻译 API |
| 腾讯翻译君 | [console.cloud.tencent.com/cam/capi](https://console.cloud.tencent.com/cam/capi) | 获取 SecretId/SecretKey |
| Bing/Azure | [portal.azure.com](https://portal.azure.com/) | 创建 Translator 资源 |
| DeepLX | [DeepLX GitHub](https://github.com/OwO-Network/DeepLX) | 本地部署服务 |

### 切换方式
1. **快捷键**：`Alt+Shift+S` 快速切换
2. **命令面板**：`Ctrl+Shift+P` → "切换翻译服务"

### 切换界面

## 技术实现要点

### VSCode API 使用
| 功能 | API | 说明 |
|------|-----|------|
| 文件创建监听 | `vscode.workspace.onDidCreateFiles` | 监听文件创建事件 |
| 非英文检测 | `containsNonEnglish()` | 检测非 ASCII 字符 |
| 用户选择格式 | `vscode.window.showQuickPick` | 弹出选择框 |
| 文件重命名 | `vscode.workspace.fs.rename` | 重命名文件/目录 |
| 文件删除 | `vscode.workspace.fs.delete` | 删除文件/目录 |
| 获取选中文本 | `editor.document.getText(editor.selection)` | 获取编辑器选中内容 |
| 替换文本 | `editor.edit(editBuilder => editBuilder.replace(...))` | 替换选中内容 |
| 关闭匹配文件窗口 | 遍历 `vscode.window.visibleTextEditors` 找到匹配的文件并关闭 | 只关闭被删除的文件窗口 |
| 状态栏显示 | `vscode.window.setStatusBarMessage` | 显示临时状态信息 |

## 架构设计

### 模块职责
| 模块 | 职责 | 说明 |
|------|------|------|
| `extension.ts` | 插件入口 | 整合所有功能，注册命令和事件监听 |
| `translator.ts` | 翻译器 | 统一翻译接口，管理翻译服务降级 |
| `namingConvention.ts` | 命名格式转换 | 8种命名格式的转换逻辑（6种文件翻译 + 2种选中文本翻译额外格式） |
| `chineseDetector.ts` | 非英文字符检测 | 检测非英文字符 |
| `undoManager.ts` | 撤回管理 | 1分钟缓存机制，管理撤回记录 |
| `config.ts` | 配置管理 | 读写插件配置项 |
| `services/index.ts` | 服务接口 | 定义翻译服务接口和类型 |
| `services/pinyin.ts` | 拼音服务 | 作为翻译失败的降级方案 |
| `services/openai.ts` | OpenAI服务 | 支持自定义API端点 |
| `services/google.ts` | Google翻译服务 | 免费但有调用限制 |
| `services/bing.ts` | Bing翻译服务 | 需要Azure账号 |
| `services/deeplx.ts` | DeepLX服务 | 需要本地部署 |
| `services/baidu.ts` | 百度翻译服务 | 需要百度开放平台账号 |
| `services/tencent.ts` | 腾讯翻译服务 | 需要腾讯云账号 |

### 核心流程
```
1. 文件路径翻译流程：
   用户新建文件 → onDidCreateFiles事件 → 检测非英文 → 选择格式 → 翻译 → 重命名

2. 选中文本翻译流程：
   用户选中文本 → Alt+Shift+T → 检测非英文 → 选择格式 → 翻译 → 替换

3. 撤回流程：
   用户按Alt+Shift+Z → 查找撤回记录 → 确认删除 → 关闭窗口 → 删除文件

4. 翻译服务降级流程：
   按优先级顺序尝试翻译服务 → 失败 → 降级到下一个服务 → 所有服务失败 → 降级到拼音
```

## 错误处理

| 场景 | 处理方式 |
|------|----------|
| 翻译API调用失败 | 降级为拼音翻译 |
| 目标文件已存在 | 自动添加后缀（如 test_1.test.js）并提示 |
| 撤回超时（超过1分钟） | 提示"撤回已过期" |
| 未配置API Key | 切换时提示需要配置 |

### 调试配置
项目已配置好 `.vscode/launch.json` 和 `.vscode/tasks.json`，按 F5 即可启动调试。

### 打包插件
```bash
# 需要 Node.js >= 20
npm install -g @vscode/vsce
vsce package

# 如果 Node.js 版本较低，使用
npx vsce@2.11.0 package
```

### 文档开发
```bash
# 启动文档开发服务器
npm run docs:dev

# 构建文档
npm run docs:build

# 预览文档
npm run docs:preview
```
