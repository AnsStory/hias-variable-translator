# 变量翻译助手 - VSCode 插件

> **当前版本**：v0.1.14 | **状态**：已完成 ✅

## 功能概述

一款 VSCode 插件，自动检测并翻译任意非英文字符为英文（中文、日文、韩文、俄文等），支持文件路径翻译和选中文本翻译。

### 新增功能（v0.1.14）
- 所有翻译服务添加 10 秒请求超时控制
- 全局翻译超时保护（10秒），超时后自动降级为拼音
- OpenAI 支持自定义 baseUrl 和 model
- DeepLX 添加异步健康检查机制（60 秒缓存）
- DeepLX 健康检查竞态保护（`_healthCheckPromise`）
- 腾讯翻译支持自定义 region
- 拼音服务扩展支持日文/韩文字符
- 右键菜单：编辑器右键菜单集成翻译选中文本、翻译并复制、撤回翻译、切换服务
- 状态栏显示当前翻译服务名称
- Disposable 泄漏修复
- 激活策略为 `onStartupFinished`（确保文件事件监听器可靠注册）
- categories 改为 Formatters
- ESLint 启用 `no-explicit-any` 警告
- `.vscodeignore` 排除 `*.vsix` 旧包
- Vitest 测试框架（208 个测试用例）

### 修复项（v0.1.14）
- QuickPick dispose 泄漏修复
- 文件翻译弹窗焦点抢夺修复（waitForFileReady + 缓冲延迟）
- 翻译器并发控制（`_translating` 标志 + 全局超时）
- 防御性拼音降级（null 检查 + try/catch）

## 功能1：新建/重命名文件时翻译路径

### 使用场景
用户在 VSCode 资源管理器中右键新建文件或文件夹，或重命名现有文件/文件夹，输入包含任意非英文字符（中日韩俄等）的路径，插件自动检测语言并翻译为英文。

### 操作流程
```
新建：右键新建文件/文件夹 → 输入路径 → 回车 → 选择翻译格式 → 回车 → 自动翻译创建
重命名：右键重命名文件/文件夹 → 输入新名称 → 回车 → 选择翻译格式 → 回车 → 自动翻译
```

### 示例
```
输入：用户名称/测试文件/测试.test.js

格式选择：
- camelCase:      userName/testFile/test.test.js
- PascalCase:     UserName/TestFile/Test.test.js
- snake_case:     user_name/test_file/test.test.js
- CONSTANT_CASE:  USER_NAME/TEST_FILE/TEST.test.js
- param-case:     user-name/test-file/test.test.js
- Header-Case:    User-Name/Test-File/Test.test.js
- noCase:         user name /test file /test.test.js
- noCaseCamel:    userName/testFile/test.test.js
```

### 取消操作行为
```
新建文件：按 ESC 取消 → 删除新建的文件/文件夹
重命名文件：按 ESC 取消 → 保留原文件/文件夹（不删除）
```

### 撤回功能
```
按 Alt+Shift+Z → 删除翻译后的文件/目录 → 自动清理空目录 → 关闭该文件的编辑器窗口
```

**撤回规则**：
- 撤回操作会直接删除翻译后的文件，而不是恢复到原始非英文路径
- 仅清理翻译时创建的目录：通过对比翻译前后的路径判断，相同前缀段（如 `src/`）属于用户原有目录，绝不清理
- 只关闭被删除文件的编辑器窗口，不会关闭其他已打开的文件
- **1分钟内可撤回**，1分钟后自动清空撤回缓存

> 例如：在 `src/` 下输入 `你好/世界/美好.test.js`，翻译为 `src/hello/world/beautiful.test.js` 后撤回，会删除 `hello/world/` 和 `hello/`，但 `src/` 无论是否为空都会保留。

### 文件名冲突处理
```
场景：目标文件已存在（如 test.test.js 已存在）
处理：自动添加后缀（如 test_1.test.js）并提示用户
```

### 翻译失败降级
```
场景：翻译API调用失败（网络错误、配额用完等）
处理：按优先级降级到下一个服务 → 所有服务失败 → 自动降级为拼音翻译
全局翻译超时（10秒）→ 自动降级为拼音转换
```

### 配置项
此功能可在设置中开启/关闭：
```json
{
  "variableTranslator.enableFileTranslation": true
}
```

## 剪贴板功能设计（v0.1.12）

### 需求背景
用户希望翻译后能快速获取多种命名格式的结果，方便在不同场景下使用。

### 功能设计

#### 配置项说明
```json
{
  "variableTranslator.copyToClipboard": {
    "type": "boolean",
    "default": false,
    "description": "翻译完成后自动将结果复制到剪贴板（需配合 clipboardFormats 使用）"
  },
  "variableTranslator.clipboardFormats": {
    "type": "array",
    "items": {
      "type": "string",
      "enum": [
        "camelCase",
        "PascalCase",
        "snake_case",
        "CONSTANT_CASE",
        "param-case",
        "Header-Case",
        "Capital Case",
        "no case"
      ]
    },
    "default": [],
    "description": "复制到剪贴板的命名格式（支持多选，按顺序逐条写入剪贴板历史）"
  }
}
```

#### 配置逻辑
| copyToClipboard | clipboardFormats | 行为 |
|-----------------|------------------|------|
| false | 任意 | 只翻译，不复制 |
| true | 有值 | 翻译后复制：用户选择的格式排第一 + clipboardFormats 其余格式 |
| true | 空数组 | 翻译后只复制用户选择的单个格式 |

### 场景1：选中文本翻译

#### 快捷键：Alt+Shift+T（翻译并替换）
```
1. 选中文本 → 按 Alt+Shift+T
2. 弹出格式选择框 → 用户选择格式
3. 翻译并替换选中文本
4. 如果 clipboardFormats 有值：
   - 重排 clipboardFormats（用户选择的排第一）
   - 按重排后的格式依次复制到剪贴板
```

#### 快捷键：Alt+Shift+C（翻译并复制，不替换）
```
1. 选中文本 → 按 Alt+Shift+C
2. 弹出格式选择框 → 用户选择格式
3. 翻译（不替换原文）
4. 如果 copyToClipboard=true：
   - 如果 clipboardFormats 有值：重排后依次复制
   - 如果 clipboardFormats 为空：只复制用户选择的格式
5. 如果 copyToClipboard=false：不复制
```

#### 剪贴板写入示例
```
选中：用户名称
配置：["camelCase", "snake_case", "CONSTANT_CASE"]
用户选择格式：snake_case

执行 Alt+Shift+C 后：
剪贴板顺序重排：["snake_case", "camelCase", "CONSTANT_CASE"]（用户选择的排第一）
- 剪贴板历史第1条：user_name
- 剪贴板历史第2条：userName
- 剪贴板历史第3条：USER_NAME
- 当前剪贴板：USER_NAME（最后写入的）
```

#### 剪贴板顺序重排规则
```typescript
// 用户选择的格式排到第一，其余保持原序
function reorderFormats(formats: string[], selected: string): string[] {
  const rest = formats.filter(f => f !== selected);
  return [selected, ...rest];
}
```

### 技术实现要点

#### VSCode API
| 功能 | API | 说明 |
|------|-----|------|
| 写入剪贴板 | `vscode.env.clipboard.writeText(text)` | 每次调用会覆盖当前剪贴板，但剪贴板历史工具可记录 |
| QuickPick 多选 | `vscode.window.showQuickPick(items, {canPickMany: true})` | 支持多选格式 |

#### 剪贴板历史实现
```typescript
// 逐条写入剪贴板（利用剪贴板历史工具记录）
for (const format of clipboardFormats) {
  const result = convertNaming(text, format);
  await vscode.env.clipboard.writeText(result);
}
// 最终保留最后一个格式在剪贴板
```

## 配置项

### 新增配置（已完成）

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `variableTranslator.copyToClipboard` | boolean | false | 翻译后是否自动复制到剪贴板 |
| `variableTranslator.clipboardFormats` | string[] | [] | 复制到剪贴板的命名格式（支持多选） |

#### clipboardFormats 可选值
- `camelCase` - 小驼峰
- `PascalCase` - 大驼峰
- `snake_case` - 下划线
- `CONSTANT_CASE` - 常量格式
- `param-case` - 连字符
- `Header-Case` - 头部连字符
- `no case` - 空格分隔
- `originalValue` - 原始翻译前的文本

#### 使用示例
```json
{
  "variableTranslator.copyToClipboard": true,
  "variableTranslator.clipboardFormats": [
    "camelCase",
    "originalValue",
    "PascalCase",
    "no case",
    "snake_case",
    "CONSTANT_CASE",
    "param-case",
    "Header-Case",
    "Capital Case",,
    "CONSTANT_CASE"
  ]
}
```

**效果**：翻译"用户名称"后，剪贴板历史中会依次包含：
- `用户名称`（原始值）
- `userName`（camelCase）
- `user_name`（snake_case）
- `USER_NAME`（CONSTANT_CASE）

最终剪贴板保留最后一个格式（用户选择的格式会排到第一位）

## 架构设计

### 模块职责
| 模块 | 职责 | 说明 |
|------|------|------|
| `extension.ts` | 插件入口 | 整合所有功能，注册命令和事件监听 |
| `translator.ts` | 翻译器 | 统一翻译接口，管理翻译服务降级 |
| `clipboard.ts` | 剪贴板工具 | 处理翻译结果的剪贴板复制功能 |
| `namingConvention.ts` | 命名格式转换 | 9种命名格式的转换逻辑（6种文件翻译 + 2种选中文本翻译额外格式 + originalValue） |
| `chineseDetector.ts` | 非英文字符检测 | 检测非英文字符 |
| `undoManager.ts` | 撤回管理 | 1分钟缓存机制，管理撤回记录 |
| `config.ts` | 配置管理 | 读写插件配置项 |
| `services/utils.ts` | 共享工具 | fetchWithTimeout 超时控制，所有服务共用 |
| `services/index.ts` | 服务接口 | 定义翻译服务接口和类型 |
| `services/pinyin.ts` | 拼音服务 | 降级方案，支持中日韩字符拼音转换 |
| `services/openai.ts` | OpenAI服务 | 支持自定义 baseUrl/model，兼容第三方 API |
| `services/google.ts` | Google翻译服务 | 官方 Cloud Translation API，需 API Key |
| `services/bing.ts` | Bing翻译服务 | 需要Azure账号 |
| `services/deeplx.ts` | DeepLX服务 | 本地部署，自动健康检查，支持自定义 baseUrl |
| `services/baidu.ts` | 百度翻译服务 | 需要百度开放平台账号 |
| `services/tencent.ts` | 腾讯翻译服务 | 支持自定义区域 |

### 核心流程
```
1. 文件路径翻译流程：
   用户新建文件 → onDidCreateFiles事件 → 检测非英文 → 选择格式 → 翻译 → 重命名 → 复制到剪贴板

2. 选中文本翻译流程：
   用户选中文本 → Alt+Shift+T → 检测非英文 → 选择格式 → 翻译 → 替换 → 复制到剪贴板

3. 翻译并复制流程：
   用户选中文本 → Alt+Shift+C → 检测非英文 → 选择格式 → 翻译 → 复制到剪贴板（不替换）

4. 撤回流程：
   用户按Alt+Shift+Z → 查找撤回记录 → 确认删除 → 关闭窗口 → 删除文件

5. 翻译服务降级流程：
   按优先级顺序尝试翻译服务 → 失败 → 降级到下一个服务 → 所有服务失败 → 降级到拼音
```

## 错误处理

| 场景 | 处理方式 |
|------|----------|
| 翻译API调用失败 | 降级为拼音翻译 |
| 目标文件已存在 | 自动添加后缀（如 test_1.test.js）并提示 |
| 撤回超时（超过1分钟） | 提示"撤回已过期" |
| 未配置API Key | 切换时提示需要配置 |
