打印需求计划 ​

当前版本：v0.1.16 | 状态：已完成 ✅

功能概述 ​

功能1：插入 console.log ​

选中文件中的部分内容（支持多选），按下快捷键 Ctrl+Alt+L 后，在下一行添加 console.log 语句。

支持 snippet 语法（光标占位符），可自定义光标跳转位置。

功能2：批量管理 console.log ​

  • Ctrl+Alt+D - 删除文件中与配置格式一致的 console.log
  • Ctrl+Alt+C - 注释文件中与配置格式一致的 console.log
  • Ctrl+Alt+U - 取消注释文件中与配置格式一致的 console.log

重要：仅处理与配置的 consoleLogTemplate 格式一致的 console.log，不一致的不进行处理。

功能3：插入后自动复制到剪贴板 ​

插入 console.log 后，自动将模板中指定的部分复制到剪贴板。

使用与 consoleLogTemplate 相同的变量语法配置要复制的内容。

配置示例 ​

json
// 普通模式
"variableTranslator.consoleLogTemplate": "${value}, `${name}`"

// Snippet 模式：插入后光标停在 name 位置
"variableTranslator.consoleLogTemplate": "${value}, `${1:name}`$0"

// Snippet 模式：插入后光标停在 value 位置
"variableTranslator.consoleLogTemplate": "${1:value}, `${name}`$0"


// 启用剪贴板复制：复制 log_ 前缀后的变量名
"variableTranslator.consoleLogTemplate": "${value}$1, `${name:log_,}`$0",
"variableTranslator.consoleLogCopyToClipboard": true
"variableTranslator.consoleLogClipboardPattern": "${name:log_,}"

Snippet 模式说明 ​

  • 如果模板中包含 $1、${1:...} 等语法，则启用 snippet 模式
  • 单选和多选均支持 snippet 模式
  • 使用 WorkspaceEdit + SnippetTextEdit API 实现多位置 snippet 插入
  • 插入后按 Tab 键可在各光标位置间跳转

技术实现 ​

1. 插入 console.log ​

核心逻辑 ​

  1. 获取选中的文本内容（支持多选）
  2. 读取模板配置
  3. 判断是否启用 snippet 模式
  4. 解析模板变量（${value}、${name}、${name:prefix,suffix}）
  5. 使用 AST 解析确定正确的插入位置
  6. 根据模式插入：
    • Snippet 模式：使用 WorkspaceEdit + SnippetTextEdit 支持多位置 snippet
    • 普通模式：使用 editor.edit() 直接插入文本
  7. 处理缩进（保持与当前行相同的缩进）

插入位置计算（语句锚定模型） ​

定位逻辑已于 v0.1.15 从「16 种表达式场景枚举」重构为「语句锚定模型」，实现见 src/print/locator.ts，设计全过程见 打印定位逻辑重构计划.md。

使用 acorn + @sveltejs/acorn-typescript 解析代码生成 AST，不再枚举表达式形状，而是三步纯函数定位：

  1. 命中目标节点：以选区 [startOffset, endOffset]（offset 而非行号，消除同行同名歧义）找到被选区完整包含且区间最小的节点，成员路径整体优先，并解包类型断言/括号/可选链。
  2. 上爬锚语句 + 判角色：沿父链上爬到最近的语句容器（Program/BlockStatement/SwitchCase/StaticBlock/TSModuleBlock），确定目标在锚语句中的角色。
  3. 决策 placement：由角色查表输出插入位。
Placement触发角色语义
AFTER_STMT（默认）产生位：变量声明、赋值左值变量在此获得值，插到语句后
BLOCK_HEAD函数参数 / 循环头声明参数/循环变量在块内才可见，插到块首行
BEFORE_STMT使用位：条件、return 实参、游离读取打印已存在的值，插到语句前
SKIP类型层（枚举成员/接口属性等）、ClassBody 直属类字段无运行时值或放语句非法，静默不插入

无块体形态（() => expr、无花括号 if/for）经 normalize 补块（wrap-expr-return / wrap-stmt / 空块展开）后再定位；多选批量以原始 code 坐标原子编辑、offset 降序应用、应用前重叠自检，杜绝坐标漂移。

角色判定优先级 ​

determineRole 按以下优先级判定目标角色，decidePlacement 据此输出 placement（正交、可独立单测，不依赖规则先后）：

优先级角色placement
1TYPE_LAYER（类型层）SKIP
2PARAM（函数参数，含构造器参数属性）BLOCK_HEAD
3LOOP_HEAD（循环头声明）BLOCK_HEAD
4PRODUCE（产生位）AFTER_STMT
5USE（使用位）BEFORE_STMT

ClassBody 直属类字段（config / #private）锚容器为 ClassBody 而非语句容器，统一 SKIP（类体直属层放语句非法）。

AST 解析流程 ​

1. 获取文件扩展名
2. 判断是否需要提取 script（Vue/HTML/Svelte/Astro），按选区所在行选定 script 块
3. 使用 acorn + @sveltejs/acorn-typescript 解析代码
4. 调整 AST 位置：script 块内偏移 rebase 为全文档绝对 offset，并按 VSCode 行规则重算 loc
5. resolveTargetNode：选区 offset → 目标节点（区间最小 / 成员路径整体 / 解包下钻）
6. climbToAnchor + determineRole + decidePlacement：锚语句/容器 → 角色 → placement
7. computeInsertionAnchor：输出结构化 offset 锚点（含缩进锚点、normalize 补块描述）
8. planInsertions：多选规划为原始坐标编辑，重叠自检 + 同 offset 稳定序

2. 批量管理 console.log ​

核心逻辑 ​

  1. 读取整个文件内容
  2. 根据配置的模板格式，构建匹配正则表达式
  3. 遍历每一行，检查是否匹配模板格式
  4. 根据命令执行相应操作：
    • 删除：移除匹配的行
    • 注释：在行首添加 //
    • 取消注释：移除行首的注释符号
  5. 应用修改到文件

核心函数 ​

handler.ts

  • handleInsertConsoleLog() - 插入 console.log（支持多选和 snippet）
  • handleDeleteConsoleLog() - 批量删除 console.log
  • handleCommentConsoleLog() - 批量注释 console.log
  • handleUncommentConsoleLog() - 批量取消注释 console.log

config.ts

  • parseConsoleLogTemplate(value, template) - 解析模板变量
  • hasSnippetSyntax(template) - 检查模板是否包含 snippet 语法
  • buildConsoleLogRegex(template) - 构建匹配正则表达式

ast.ts（解析基元）

  • parseCode(code, selectionLine, fileExtension) - 解析代码生成 AST，模板文件先提取 script 并 rebase offset
  • isTemplateExtension(ext) - 判断是否为需提取 script 的模板文件（Vue/HTML/Svelte/Astro）
  • offsetToLine(code, offset) - 按 VSCode 行规则将 offset 换算为 0-based 行号
  • walkAST(node, cb) - 深度优先遍历 AST
  • buildParentMap(ast) - 构建节点到父节点的映射表（按 ast 根节点记忆化）
  • unwrapTypeAssertion(node) - 递归解包类型断言/括号/可选链
  • getLineIndent(lineText) - 获取行缩进前缀

locator.ts（语句锚定定位器）

  • resolveTargetNode(ast, code, selection) - 选区 offset → 目标节点（区间最小 / 成员路径整体 / 解包下钻）
  • climbToAnchor(target, ast) - 上爬到最近语句容器，返回锚语句与容器
  • determineRole(target, ast) - 判定目标角色（PRODUCE/USE/PARAM/LOOP_HEAD/TYPE_LAYER）
  • decidePlacement(target, ast) - 角色 → placement（AFTER_STMT/BEFORE_STMT/BLOCK_HEAD/SKIP）
  • computeInsertionAnchor(ast, code, selection) - 输出结构化 offset 锚点（含缩进锚点与 normalize 补块描述）
  • planInsertions(ast, code, selections) - 多选规划为原始坐标编辑：重叠自检 + 同 offset 稳定序 + normalize/skip/fallback 分流

注意事项 ​

  • 功能开关：默认禁用，需在设置中开启 enableConsoleLog
  • 缩进处理：自动读取当前行的缩进，并应用到新插入的行
  • 多选支持：支持同时选中多个变量，为每个变量插入对应的 console.log
  • Snippet 支持：单选和多选均支持 snippet 语法，可自定义光标跳转位置
  • 模板灵活性：用户可自定义输出格式，支持反引号、单引号、双引号包裹
  • 匹配准确性：仅处理与配置模板格式一致的 console.log，不匹配的保留不变
  • 定位模型：采用语句锚定模型（offset 节点级命中 + 角色→placement 决策表），新语法零代码自动覆盖，替代原 16 场景枚举
  • TypeScript 支持：支持类型断言、非空断言、可选链、参数属性等特殊语法
  • 文件类型支持：支持 JS/TS/Vue/HTML/JSX/TSX/Svelte/Astro 文件
  • 补块归一化：箭头表达式体 / 无花括号块经 normalize 统一补块（wrap-expr-return / wrap-stmt / 空块展开）后再定位

用户配置示例（settings.json） ​

json
{
  // 启用 console.log 功能
  "variableTranslator.enableConsoleLog": true,

  // ========== 普通模式配置 ==========

  // 默认配置：反引号包裹
  "variableTranslator.consoleLogTemplate": "${value}, `${name}`"

  // 单引号包裹
  "variableTranslator.consoleLogTemplate": "${value}, '${name}'"

  // 双引号包裹
  "variableTranslator.consoleLogTemplate": "${value}, \"${name}\""

  // 带前后缀
  "variableTranslator.consoleLogTemplate": "${value}, ${name:前缀,后缀}"

  // ========== Snippet 模式配置 ==========

  // 插入后光标停在 name 位置
  "variableTranslator.consoleLogTemplate": "${value}, `${1:name}`$0"

  // 插入后光标停在 value 位置
  "variableTranslator.consoleLogTemplate": "${1:value}, `${name}`$0"

  // 多光标跳转
  "variableTranslator.consoleLogTemplate": "'❁', ${value}$1, `${name:前,后}`$0"

  // ========== 剪贴板复制配置 ==========

  // 启用剪贴板复制功能
  "variableTranslator.consoleLogCopyToClipboard": true,

  // 复制 log_ 前缀后的变量名（如选中 params，复制 params）
  "variableTranslator.consoleLogClipboardPattern": "${name:log_,}",

  // 复制整个带反引号的 name（如选中 params，复制 `params`）
  "variableTranslator.consoleLogClipboardPattern": "${name:`",

  // 复制选中的原始文本
  "variableTranslator.consoleLogClipboardPattern": "${value}"
}

模板语法速查 ​

语法说明示例
${value}选中的原始文本obj
${name}原始文本（不添加引号）obj
${name:前,后}前缀+原始文本+后缀前obj后
$1, $2, ...snippet 光标位置按 Tab 跳转
$0最终光标位置最后停留
${1:placeholder}带默认值的占位符选中可编辑

注意：consoleLogClipboardPattern 使用与 consoleLogTemplate 相同的变量语法，但不支持 snippet 语法（$1、$0 等）
