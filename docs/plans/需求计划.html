变量翻译助手 - VSCode 插件 ​

功能概述 ​

一款 VSCode 插件，自动检测并翻译任意非英文字符为英文（中文、日文、韩文、俄文等），支持文件路径翻译和选中文本翻译。

功能1：新建文件/文件夹时翻译路径 ​

使用场景 ​

用户在 VSCode 资源管理器中右键新建文件或文件夹，输入包含任意非英文字符（中日韩俄等）的路径，插件自动检测语言并翻译为英文。

操作流程 ​

右键新建文件/文件夹 → 输入路径 → 回车 → 选择翻译格式 → 回车 → 自动翻译创建

示例 ​

输入：用户名称/测试文件/测试.test.js

格式选择：
- camelCase:      userName/testFile/test.test.js
- PascalCase:     UserName/TestFile/Test.test.js
- snake_case:     user_name/test_file/test.test.js
- CONSTANT_CASE:  USER_NAME/TEST_FILE/TEST.test.js
- param-case:     user-name/test-file/test.test.js
- Header-Case:    User-Name/Test-File/Test.test.js
- noCase:         user name /test file /test.test.js
- noCaseCamel:    userName/testFile/test.test.js

撤回功能 ​

按 Alt+Shift+Z → 删除翻译后的文件/目录 → 关闭该文件的编辑器窗口

撤回规则：

  • 撤回操作会直接删除翻译后的文件，而不是恢复到原始非英文路径
  • 只关闭被删除文件的编辑器窗口，不会关闭其他已打开的文件
  • 1分钟内可撤回，1分钟后自动清空撤回缓存

文件名冲突处理 ​

场景：目标文件已存在（如 test.test.js 已存在）
处理：自动添加后缀（如 test_1.test.js）并提示用户

翻译失败降级 ​

场景：翻译API调用失败（网络错误、配额用完等）
处理：自动降级为拼音翻译

配置项 ​

此功能可在设置中开启/关闭：

json
{
  "variableTranslator.enableFileTranslation": true
}

功能2：编辑器中选中文本翻译 ​

使用场景 ​

用户在任意文件中编写非英文字符（中日韩俄等），选中后通过快捷键自动检测语言并翻译替换为英文。

操作流程 ​

选中文本 → 按下 Alt+Shift+T → 选择翻译格式 → 回车 → 替换为英文

示例 ​

选中：用户名称
按下：Alt+Shift+T

格式选择：
- camelCase:      userName
- PascalCase:     UserName
- snake_case:     user_name
- CONSTANT_CASE:  USER_NAME
- param-case:     user-name
- Header-Case:    User-Name
- Capital Case:   User Name
- no case:        user name

撤回功能 ​

使用 VSCode 自带的撤销功能：Ctrl+Z

命名格式说明 ​

文件翻译格式 ​

格式示例说明
camelCaseuserName小驼峰，首字母小写
PascalCaseUserName大驼峰，首字母大写
snake_caseuser_name下划线分隔，全小写
CONSTANT_CASEUSER_NAME下划线分隔，全大写
param-caseuser-name连字符分隔，全小写
Header-CaseUser-Name连字符分隔，首字母大写

选中文本翻译额外格式 ​

格式示例说明
Capital CaseUser Name首字母大写，空格分隔
no caseuser name全小写，空格分隔

快捷键说明 ​

快捷键功能说明
Alt+Shift+T翻译选中文本编辑器中选中非英文文本后翻译
Alt+Shift+Z撤回文件翻译删除翻译后的文件/目录（1分钟内有效）
Alt+Shift+D切换文件翻译开关开启/关闭文件路径翻译功能
Alt+Shift+S切换翻译服务选择不同的翻译服务

核心流程 ​

1. 文件路径翻译流程：
   用户新建文件 → onDidCreateFiles事件 → 检测非英文 → 选择格式 → 翻译 → 重命名

2. 选中文本翻译流程：
   用户选中文本 → Alt+Shift+T → 检测非英文 → 选择格式 → 翻译 → 替换

3. 撤回流程：
   用户按Alt+Shift+Z → 查找撤回记录 → 确认删除 → 关闭窗口 → 删除文件

4. 翻译服务降级流程：
   按优先级顺序尝试翻译服务 → 失败 → 降级到下一个服务 → 所有服务失败 → 降级到拼音

错误处理 ​

场景处理方式
翻译API调用失败降级为拼音翻译

未配置API Key切换时提示需要配置

使用说明 ​

安装依赖 ​

bash
npm install --registry https://registry.npmmirror.com

编译项目 ​

bash
# 开发编译
npm run compile

# 生产构建（带代码混淆）

  1. 在 VSCode 中打开项目
  2. 按 F5 启动调试（会自动编译并打开新窗口）
